# Default Plan Template

Write a concise Markdown plan for the requested goal.

Use this structure:

1. `Goal`: one sentence describing the end state.
2. `Scope`: concrete files, modules, commands, or behavior areas that are in scope.
3. `Steps`: ordered implementation steps with observable outcomes.
4. `Verification`: commands, tests, or manual checks that prove the plan is complete.
5. `Risks`: unresolved assumptions or dependencies that could affect execution.

Keep the plan actionable. Use the suggested project plan directory when the caller provides one.
